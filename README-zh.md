# task-spec

防止 AI 在思考之前就动手——通过在每一次任务上运行对抗性质控循环。

## 问题

LLM 有两个毛病。一个在动手前，一个在动手后。

现代 AI 像一个从不检查自己工作的聪明人。给你的答案看起来都对——
自信、有条理、合理——但它从来不停下来想想这答案到底对不对。

原因比"懒"深刻。LLM 生成文本的方式是一次预测一个词，每次选
统计上最可能的下一个。它的第一直觉永远是最可能的路径，不是
最谨慎的那条。一个正确答案和一个看似合理但错误的答案，在统计
数字里长得一样。模型从内部无法区分。

这体现在两件事上：

**动手前——想得浅。** 脑海里冒出第一个方向就走。不找替代方案，
不问"这真的是该解决的问题吗"。把"有个答案"和"答案是对的"
当成一回事。

**动手后——不检查。** "做完了"等于"做好了"。不会发现前后矛盾，
不会拿结果跟初衷比对，不会问漏了什么。

更好的训练数据修不了这两个毛病。数据只能教会模型正确的产出
*长什么样*——学不到产出之前的思考。人类的犹豫、被放弃的第一
版、落笔前脑子里的自我检查——在最终结果里全不可见。模型只看到
输出，从来看不到过程。

## 能修吗？

这是架构性的。模型生成文本的方式——一个接一个挑统计上最可能的
词——从来不问"这对吗"。换训练数据、换 prompt 写法，都加不进去
这个问题。

**模型层**：研究在探索能中途暂停、评估、修正的架构。但这是前沿，
今天还不能指望。

**系统层**：两个毛病的性质不同。

验证是容易的那个。"输出和 spec 对得上吗"是客观问题。从外面就能
强制执行——合约、核对清单、证据。

思考更难。"这方案对吗""这够好了吗"需要一个模型没有的东西：指向
"更好"的内部罗盘。模型只朝向"最可能"，从不朝向"更好"。它不是
懒才停在浅层——是分不清哪个方向能走得更深。

人类的对应概念是态度。一个想把事做好的人，自然会探索、比较、质疑、
推翻重来。模型没有对等的驱动力。它不会"想"任何事。

所以两个毛病需要不同的对策。验证能用合约强制执行。思考只能靠结构
去推——逼模型走一遍高态度的人自然而然会做的步骤。

## 答案

task-spec 用五个阶段分别应对两个维度：

**探索 → 定规 → 挑战 → 执行 → 审查**

**探索**——画出解空间。理解问题、考虑方向、排除死胡同。先别锁。

**定规**——锁定方向。定义可测量的验收标准、预判风险、写下推理依据。
spec 成为绑定合约：执行照它推进，审查依它判定。

**挑战**——对抗性审查。五个问题：边界情况、错误假设、过度设计、
风险、解空间是否遍历完整。非平凡任务委派给 subagent。没过这关的
spec 不能进入执行。

**执行**——严格照 spec 走。每一步都必须可追溯到 spec。遇到未覆盖
的情况，暂停，回到探索。

**审查**——逐条验收标准对照实际产出，附证据。全过？完成。有没过的？
先分析失败是否真实，再回到探索启动新循环。

五个阶段形成两层循环。**内循环**（探索 → 定规 → 挑战）在动手前
打磨方案——轻微缺陷回定规，重大缺陷回探索。**外循环**（执行 → 审查
→ 探索）在动手后验证结果——审查不通过就重启。

每次的 spec 写入 `.task_spec/<slug>.md`，留下审计链。
